# Security Policy

## Supported Versions

Security fixes are provided for the latest released version.

| Version | Supported |
| --- | --- |
| Latest | :white_check_mark: |
| Older releases | :x: |

## Reporting a Vulnerability

Please do not open public GitHub issues for security vulnerabilities.

Use one of the following private channels:

1. GitHub Security Advisories (preferred): open a private report in the repository Security tab.
2. Email the maintainers directly if private reporting is not available.

Include as much detail as possible:

- A clear description of the issue and impact
- Steps to reproduce or a proof of concept
- Affected versions and environments
- Any suggested mitigation

## Response Process

- Initial acknowledgment target: within 3 business days
- Triage and severity assessment: as soon as possible
- Fix and coordinated disclosure timeline: based on severity and exploitability

We will keep reporters informed throughout triage and remediation.

## Disclosure

Please allow time for a fix before public disclosure. After a fix is released,
we will publish security details in release notes or advisories as appropriate.
